# Roles Overview

OpsFlow uses role-based access control (RBAC) to manage
permissions across the platform.

## Default Roles

OpsFlow includes the following default roles:

| Role | Description |
|---|---|
| Employee | Basic operational access |
| Supervisor | Team management and approvals |
| Administrator | Full platform configuration and security management |

## Permission Categories

Permissions control access to:

- work orders
- scheduling
- reporting
- incident management
- workflow automation
- user administration
- API configuration

## Best Practices

- Assign users the minimum permissions required.
- Review permissions quarterly.
- Use custom roles for specialized workflows.
